# Security Policy

## 🚨 支持的版本

我们只对以下版本提供安全更新：

| Version | Supported          |
| ------- | ------------------ |
| 0.5.x   | ✅ Latest          |
| < 0.5   | ❌ Not supported   |

> 请确保您使用的是最新版本以获得所有安全修复。

---

## 🔒 报告安全漏洞

我们非常重视安全问题。如果您发现了安全漏洞，请通过以下方式报告：

### 📧 报告方式

**强烈推荐：通过邮件报告**

请发送邮件至：**liuhuanxi-oss@users.noreply.github.com**

邮件标题格式：`[SECURITY] 店赢OS - 简要描述问题`

### 📝 报告内容

为了帮助我们快速评估和复现问题，请包含：

1. **问题类型**：XSS、CSRF、注入、数据泄露等
2. **受影响版本**：问题出现的版本号
3. **复现步骤**：详细的攻击步骤
4. **潜在影响**：如果被利用会造成什么影响
5. **您的信息**（可选）：您的邮箱或GitHub用户名，以便我们致谢

### ⚡ 响应承诺

- **48小时内**：确认收到您的报告
- **7天内**：提供初步评估和修复计划
- **持续沟通**：在修复过程中保持更新

---

## 🔐 负责任披露政策

我们遵循负责任的安全披露原则：

1. **私下报告**：请勿在公开渠道（如GitHub Issues、社交媒体）讨论安全问题
2. **协作解决**：我们愿意与您合作验证和修复问题
3. **及时修复**：尽快发布安全更新
4. **公开感谢**：修复后，将在下方致谢贡献者

### ❌ 不允许的行为

- 未经允许公开讨论安全问题
- 利用漏洞进行非授权测试
- 访问或修改其他用户数据
- 进行可能影响服务可用性的测试
