## 一、系统概述

### 1.1 产品定位

**店赢OS** 是一款从零设计的AI原生门店操作系统，让一个人可以运营无限门店。

### 1.2 核心目标

- 降低门店运营门槛
- 提升运营效率
- 实现AI自主决策

## 三、核心模块说明

### 3.1 AI虚拟店长

负责整体门店运营决策，调用各Agent协同工作。

**核心能力**：
- 自然语言理解
- 意图识别
- 任务分发
- 策略制定

### 3.2 多Agent协同

| Agent | 职责 | 核心能力 |
|-------|------|---------|
| 线索Agent | 发现问题 | 评价监控、趋势分析、风险预警 |
| 方案Agent | 分析方案 | 根因分析、方案生成、知识匹配 |
| 交付Agent | 执行任务 | 任务下发、进度追踪、效果验证 |
| 运营Agent | 持续优化 | 知识沉淀、策略迭代、经验迁移 |

### 3.3 核心业务模块

| 模块 | 功能 |
|------|------|
| review-handler | 评价智能管理 |
| pricing-engine | 动态定价引擎 |
| digital-twin | 门店数字孪生 |
| knowledge-transfer | 跨店知识迁移 |
| store-clone | 门店克隆 |
| lifecycle-drive | 客户生命周期自动驾驶 |

## 四、数据架构

### 4.1 数据流转

```
外部数据 → 数据汇聚 → 智能分析 → Agent协同 → 指令输出
    ↑           ↓           ↓           ↓           ↓
    └───────────┴───────────┴───────────┴───────────┘
                    反馈闭环
```

### 4.2 信任等级机制

| 等级 | 分数 | 处理方式 |
|------|-----|---------|
| A级 | 95-100 | AI全自动执行 |
| B级 | 85-94 | AI执行+通知 |
| C级 | 60-84 | AI生成+确认 |
| D级 | 40-59 | AI生成+优先人工 |
| E级 | <40 | 预警+强制人工 |

## 五、技术选型

| 组件 | 选择 | 说明 |
|------|------|------|
| 前端框架 | 原生JS | 无需构建，直接运行 |
| AI平台 | Coze | 多Agent协同编排 |
| 知识库 | Coze知识库 | 原生集成 |
| 数据存储 | LocalStorage | MVP阶段足够 |
| 图标 | Emoji | 轻量美观 |

## 六、部署架构

### 6.1 前端部署

```
src/
├── index.html      # 主入口
├── css/
│   └── style.css   # 样式
└── js/
    ├── app.js      # 主逻辑
    ├── agents/     # Agent模块
    └── modules/    # 业务模块
```

### 6.2 运行方式

- 直接用浏览器打开 `index.html`
- 或使用本地服务器：
  ```bash
  python -m http.server 8080
  ```
